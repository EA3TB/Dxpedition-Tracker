📡 DXpedition Tracker

Docker / NAS Windows .exe

Web dashboard for real-time tracking of active DXpeditions. Displays the band/mode status of each expedition by cross-referencing the HRD, Swisslog, Log4OM or ADIF log against the DXCC entity database (cty.dat), and includes ARS-USB azimuth rotor control.

Two executables, one launch. The Windows application consists of two executables: DXpeditionTracker.exe (main dashboard) and rotor_tray_exe.exe (ARS-USB rotor control). You only need to launch DXpeditionTracker.exe — it automatically starts the rotor tray in the background.
Read-only. This application does not write or modify anything in HRD, Swisslog, Log4OM or ADIF logs. It only reads data to display it on the dashboard.
Important. The application loads all contacts made with the DXCC entity (country) to date, not just those made with the exact DXpedition callsign. The ✕ "just worked" cell marker serves as a reminder that this specific station has been worked on that band/mode during the active expedition.

🖥 Main dashboard view

The dashboard opens automatically in the browser when the application is launched. It is divided into five horizontal zones plus the card grid.

🗂 DXpedition cards

Cell states reflect the complete history with the DXCC entity (country), not just with the expedition callsign. For example, if you worked Norfolk Island last year with a different callsign on 20m FT8, that cell will appear as Worked or Confirmed. The ✕ marker indicates that this specific expedition has already been worked on that band/mode.

Cell states

  • C — Confirmed: Physical QSL or LoTW received with the entity on that band/mode
  • W — Worked: QSO in the log, not yet confirmed
  • ✕ — Just worked: manually marked to remember this expedition was worked. Right-click to undo
  • Empty: band/mode not worked with this entity

SP% / LP% Propagation

Real-time calculation using the Kp index (NOAA) and SFI/SSN (hamqsl.com), for the selected transmit power and active modes. The score shown is the best value across all active modes.

  • ● Green ≥70% — good propagation
  • ● Orange 40–69% — fair propagation
  • ● Red <40% — poor propagation

SP / LP Azimuths

The Short Path and Long Path values are calculated from the configured Maidenhead locator. They are clickable: clicking automatically sends that azimuth to the rotor if connected.

Reorder and manage

  • Drag cards by the header to reorder them
  • Button ↺ reloads QSOs from the log for that entity
  • Button ✕ removes the card and all its data
  • Maximum 50 simultaneous cards

🌍 cty.dat database

The cty.dat database (Big CTY) is the universal reference file in the amateur radio world. It contains all recognized DXCC entities with their prefixes, geographic coordinates, CQ zones, ITU zones and continent.

Automatic download

On startup the application automatically checks for a newer version at country-files.com. If an update is available it is downloaded without user intervention. The date and version are shown in the CTY bar of the dashboard.

Dashboard usage

  • DXCC entity/country identification from callsign
  • SP and LP azimuth calculation from the user locator
  • Distance in km to the entity
  • CQ and ITU zones for reference
  • Continent for classification
The bottom bar of the dashboard shows the cty.dat status: number of loaded entities and version date. If shown in red, the database could not be downloaded and the last cached version is being used.

📅 DX Calendar

Automatically creates and removes DXpedition cards based on the NG3K Announced DX Operations calendar.

How it works

  • Enable with the DX Calendar checkbox next to the locator
  • On every dashboard load, active DXpeditions (today's date within their announced range) are added automatically
  • Cards whose DXpedition has ended are removed automatically
  • Auto-generated cards show "till DD/MM/YYYY" next to the country name
  • Disabling the checkbox removes all auto-generated cards immediately

Manual cards

Cards added manually via + New DXpedition are never touched by DX Calendar — they are independent and must be removed manually with ✕.

Auto-generated cards are sorted among themselves by end date (soonest first); manual cards keep their position.

The maximum number of simultaneous cards is 50, to accommodate DX Calendar plus manual entries.

📡 DX Spots

Shows the latest spots for a DXpedition's callsign, fetched live from dxwatch.com.

Usage

  • Click the DX button on any card header
  • A floating window shows the last 15 spots: UTC, MHz, DE, Comments
  • Data is fetched once when opened — no auto-refresh
  • Close by clicking outside the window or the ✕

Notes

Spots reflect the exact operating callsign as reported by the cluster (e.g. FS/F4EQE), which may differ from the DXCC entity prefix shown on the card.

🎛 Mode and band filters

All filters are automatically saved to config.json and persist between sessions.

Mode filter

Checkboxes CW / SSB / FT8 / FT4 / RTTY in the control bar. Only the columns of the selected modes are shown across all cards simultaneously. A Power selector (100W / 500W / 1000W) next to the modes also affects the SP%/LP% propagation score.

Band filter

Selectable pills for the 10 available HF bands (6m, 10m, 12m, 15m, 17m, 20m, 30m, 40m, 80m, 160m). All bands are active by default. Only active band rows are shown in each card.

🔒 Validation

Before adding a DXpedition, the application checks that both the locator and the log are configured. If either is missing, a warning modal appears:

⚠️
Locator required
Please enter a valid Maidenhead locator (e.g. JN01nq) before adding a DXpedition.

The locator is required to calculate SP/LP azimuths and propagation.

The same modal appears if the log is not selected when trying to add an expedition or reload the log.

🔄 ARS-USB rotor control (EA4TX)

The rotor widget is integrated in the dashboard topbar. It supports the Yaesu GS-232A protocol, compatible with EA4TX ARS-USB and Prosistel controllers.

Topbar widget

  • Animated compass with real-time needle (500ms polling)
  • Current azimuth in degrees
  • Status: Online / Connecting / Offline
  • Button Stop — sends command S (stops rotation)
  • Exponential backoff on reconnect: 2s → 4s → … → 60s max.
  • Automatic reconnection when the ARS is powered on

Automatic movement

  • The SP° and LP° values on each card are clickable
  • Clicking sends the azimuth to the rotor automatically
  • The button flashes blue (SP) or orange (LP) while the rotor is moving
  • The chosen direction is always the shortest arc without crossing the North stop
  • Mechanical range: 0°–360°, physical stop at North (0°/360°)

🖱 Rotor Tray — Windows system tray

The rotor runs as an independent process in the system tray. This allows the rotor to keep running even if the dashboard is closed or the browser is reopened.

Status icons

Green — connected and online
Yellow — attempting to connect
Red — ARS off or offline

The status dot appears in the bottom-right corner of the DXP icon in the tray.
